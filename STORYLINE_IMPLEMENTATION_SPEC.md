# 小小侠客三主线叙事实施规格 v0.31

> 用途：给另一个 AI 直接按这份文档修改程序。  
> 项目目标：游戏最长约 3 年。每个角色拥有独立主线、独立年末 Boss、独立终局 Boss。每个月 3 个奇遇中，至少 1 个必须来自当前角色主线池。

---

## 1. 全局叙事结构

每个角色一条主线，三年结构如下：

| 时间 | 主线节奏 | Boss 定位 |
|---|---|---|
| 第 1 年 | 发现阴谋，接触外围势力 | 年末小 Boss，测试基础构筑 |
| 第 2 年 | 阴谋扩大，主角卷入核心冲突 | 年末中 Boss，开始克制玩家路线 |
| 第 3 年 | 真相揭开，阵营决裂 | 终局 Boss，高血量、多机制 |

每个月事件结构：

| 事件槽位 | 类型 | 说明 |
|---|---|---|
| 第 1 个 | 主线奇遇 | 必定从当前角色主线事件池抽取 |
| 第 2 个 | 成长奇遇 | 修炼、秘籍、金钱、装备、丹药 |
| 第 3 个 | 风险奇遇 | 普通战斗、小 Boss、代价选择 |

建议新增隐藏变量：

```js
run.storylineId = characterId;
run.mainThreat = 0;
run.storyFlags = {};
```

`mainThreat` 用于影响年末主线 Boss 难度。

| 威胁值 | 年末 Boss 加成 |
|---:|---|
| 0-2 | 无 |
| 3-5 | 血量 +10%，攻击 +5% |
| 6-8 | 血量 +20%，攻击 +10%，开场护体 10% 最大血量 |
| 9+ | 血量 +30%，攻击 +15%，开场护体 20% 最大血量，额外机制增强 |

主线奇遇常见选择影响：

| 玩家行为 | 影响 |
|---|---|
| 收集情报 | 年末 Boss 少量降属性，或少一个开场 buff |
| 放任事件 | `mainThreat +1` |
| 保护 NPC | 获得奖励，`mainThreat -1` |
| 贪图利益 | 获得金钱，`mainThreat +1` |
| 主线战斗胜利 | 获得装备、秘籍进度或 Boss 情报 |

## 2. Boss 数值基准

请结合 `IMPLEMENTATION_BALANCE_TABLES.md` 的战斗数值框架。Boss 必须显著强于普通敌人和普通小 Boss。

| 阶段 | 血量 | 内力 | 攻击 | 防御 | 命中 | 闪避 | 暴击 | 速度 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 第 1 年 Boss | 1800-2200 | 550-700 | 95-115 | 55-70 | 75-86 | 8-12 | 10-12 | 1.45-1.60 |
| 第 2 年 Boss | 3200-4200 | 900-1200 | 135-165 | 85-110 | 82-88 | 10-16 | 14-20 | 1.58-1.65 |
| 第 3 年 Boss | 6000-8000 | 1600-2300 | 190-240 | 130-170 | 90-95 | 16-22 | 18-22 | 1.72-1.82 |

Boss 设计边界：

- Boss 可以拥有强机制，但不能无边界无限叠层。
- Boss 的毒、流血、寒气、断筋、蛊都要遵守层数上限和自然衰减。
- 终局 Boss 可以有阶段机制，例如 50% 血净化一次、回血一次、开场护体一次。
- Boss 不应被单一流派完全克死；但每条主线 Boss 可以偏向克制某些玩法。

## 3. 江湖浪客主线：孤云逐浪

角色：`wanderer / 沈孤云`  
主题：江湖自由 vs 正派武盟的秩序暴政。  
主线敌人：正派武盟。  
终极 Boss：正派帮主，表面维护武林秩序，实际想借灾祸重塑江湖，铲除所有不服从自己的门派和散人。

### 年度 Boss

| 年份 | Boss ID | 名称 | 背景 | 机制定位 |
|---|---|---|---|---|
| 1 | wanderer_boss_y1 | 青旗堂主·陆闻川 | 武盟外围堂主，追杀不入盟的江湖散人 | 流血刀 + 破防 |
| 2 | wanderer_boss_y2 | 执法长老·孟天衡 | 负责清洗江湖异己，把主角列为乱党 | 高防 + 断筋 |
| 3 | wanderer_final | 武盟帮主·岳宗玄 | 以止乱为名引发浩劫，欲统一武林号令 | 护体 + 净化 + 反击 |

### Boss 数值

| Boss | 血量 | 内力 | 攻击 | 防御 | 命中 | 闪避 | 暴击 | 速度 | 特性 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---|
| 陆闻川 | 2000 | 620 | 108 | 64 | 76 | 8 | 12 | 1.48 | 每 3 回合流血 +2；直接伤害附带轻破防 |
| 孟天衡 | 3800 | 1050 | 150 | 105 | 84 | 10 | 14 | 1.58 | 断筋上限 +2；玩家速度最低可被压到 70% |
| 岳宗玄 | 7600 | 2100 | 225 | 165 | 92 | 16 | 22 | 1.72 | 开场 25% 护体；50% 血净化一次；低血反击 |

### 主线奇遇池

| 事件 ID | 名称 | 类型 | 描述 | 选择 / 效果 |
|---|---|---|---|---|
| wanderer_notice | 武盟征帖 | 选择 | 武盟命散人入册听调 | 交涉得经验；拒绝触发战斗，胜利得金钱和声望 |
| wanderer_rescue | 散人求援 | 选择 | 被武盟围捕的散人求援 | 救人得秘籍进度，`mainThreat -1`；放弃得金钱，`mainThreat +1` |

| wanderer_friend | 旧友被捕 | 选择 / 战斗 | 旧友被武盟执法堂押走 | 花钱营救或打一场主线小战 |
| wanderer_purge | 正派围剿 | 战斗 | 武盟清洗不服从者 | 胜利获得装备或内功线索 |

## 7. 程序实现建议

建议新增数据结构：

```js
DATA.storylines = {
  wanderer: {
    id: "wanderer",
    name: "孤云逐浪",
    threatName: "武盟威势",
    bosses: {
      1: { ... },
      2: { ... },
      3: { ... }
    },
    events: [ ... ]
  },
  constable: {
    id: "constable",
    name: "铁鹰入局",
    threatName: "内廷疑云",
    bosses: { ... },
    events: [ ... ]
  },
  orthodox: {
    id: "orthodox",
    name: "天衡照邪",
    threatName: "鬼教阴影",
    bosses: { ... },
    events: [ ... ]
  }
};
```

建议修改事件刷新逻辑：

```js
export function refreshEvents(run) {
  const storyEvent = sample(makeStoryEventPool(run), 1)[0];
  const growthEvent = sample(makeGrowthEventPool(run), 1)[0];
  const riskEvent = sample(makeRiskEventPool(run), 1)[0];
  run.events = [storyEvent, growthEvent, riskEvent].filter(Boolean);
}
```

建议修改年末 Boss 逻辑：

- 第 1 年 12 月触发当前角色第 1 年 Boss。
- 第 2 年 12 月触发当前角色第 2 年 Boss。
- 第 3 年 12 月触发当前角色终局 Boss。
- Boss 进入战斗前，根据 `mainThreat` 套用加成。
- 主线情报类事件可写入 `run.storyFlags`，用于削弱 Boss 某些开场机制。

## 8. 需要同步修改的文件

| 文件 | 修改内容 |
|---|---|
| `src/data/content.js` | 新增 storylines、主线 Boss、小 Boss 数据 |
| `src/systems/runSystem.js` | 每月事件池固定加入一个主线事件；年末 Boss 根据角色选择 |
| `src/systems/battleSystem.js` | 支持 Boss 特性、护体、阶段机制、威胁值加成 |
| `src/ui/render.js` | 显示当前主线名、威胁值、主线事件标签、Boss 背景 |
| `DATA_TABLES.md` | 同步新增主线、Boss、主线事件表 |
| `src/ui/render.js` 或版本配置位置 | 每次 push 前递增版本号 |
