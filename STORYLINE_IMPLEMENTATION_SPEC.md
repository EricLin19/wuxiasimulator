# 小小侠客三主线叙事实施规格 v0.36

> 用途：给另一个 AI 直接按这份文档修改程序。  
> 项目目标：游戏最长约 3 年。每个角色拥有独立主线、独立年末 Boss、独立终局 Boss。每个月 3 个奇遇中，至少 1 个必须来自当前角色主线池。

---

## 0. v0.36 新增：主线双选机制

所有主线奇遇不再只有单一选择，改为 **顺应 / 抗争** 双选：

| 选择 | 效果 |
|---|---|
| **顺应** | 接受主线势力的安排，获得奖励（金钱/属性/经验/道具），威胁值 +1~3 |
| **抗争** | 拒绝且反抗，分为三种：战斗、小Boss战、付钱 |

### 抗争类型

| 抗争类型 | 说明 |
|---|---|
| `battle` | 触发普通战斗，胜利无额外惩罚 |
| `battle_mini` | 触发小Boss级别战斗，难度更高 |
| `pay` | 花费金钱解决问题，钱不够则威胁值只加一半 |

### 实现函数

- `resolveStoryChoice(run, eventId, choice, actions)` — 处理主线双选逻辑
- `renderEventsModal` 自动识别主线事件（`type === "story"`），渲染顺应/抗争双按钮
- `renderGoalsModal` 显示威胁值说明框

### 威胁值层级

| 威胁值 | Boss 加成 | 显示标签 |
|---|---:|---|
| 0-2 | 无 | — |
| 3-5 | 血量+10%, 攻击+5% | 山雨欲来 |
| 6-8 | 血量+20%, 攻击+10% | 暗流涌动 |
| 9+ | 血量+30%, 攻击+15% | 威势压人 |

## 1. 全局叙事结构

每个角色一条主线，三年结构如下：

| 时间 | 主线节奏 | Boss 定位 |
|---|---|---|
| 第 1 年 | 发现阴谋，接触外围势力 | 年末小 Boss，测试基础构筑 |
| 第 2 年 | 阴谋扩大，主角卷入核心冲突 | 年末中 Boss，开始克制玩家路线 |
| 第 3 年 | 真相揭开，阵营决裂 | 终局 Boss，高血量、多机制 |

每个月事件结构：

| 事件槽位 | 类型 | 说明 |
|---|---|---|
| 第 1 个 | 主线奇遇 | 必定从当前角色主线事件池抽取 |
| 第 2 个 | 成长奇遇 | 修炼、秘籍、金钱、装备、丹药 |
| 第 3 个 | 风险奇遇 | 普通战斗、小 Boss、代价选择 |

建议新增隐藏变量：

```js
run.storylineId = characterId;
run.mainThreat = 0;
run.storyFlags = {};
```

`mainThreat` 用于影响年末主线 Boss 难度。

| 威胁值 | 年末 Boss 加成 |
|---:|---|
| 0-2 | 无 |
| 3-5 | 血量 +10%，攻击 +5% |
| 6-8 | 血量 +20%，攻击 +10%，开场护体 10% 最大血量 |
| 9+ | 血量 +30%，攻击 +15%，开场护体 20% 最大血量，额外机制增强 |

主线奇遇统一双选机制（v0.34）：

| 玩家行为 | 威胁值 | 奖励/代价 |
|---|---|---|
| 顺应（accept） | +1~3 | 金钱/属性/经验/道具等奖励 |
| 抗争-战斗（battle） | 不变 | 进入普通战斗 |
| 抗争-小Boss战（battle_mini） | 不变 | 进入小Boss难度战斗 |
| 抗争-付钱（pay） | 不变 | 花费指定金钱，钱不够则威胁值+一半 |

## 2. Boss 数值基准

请结合 `IMPLEMENTATION_BALANCE_TABLES.md` 的战斗数值框架。Boss 必须显著强于普通敌人和普通小 Boss。

| 阶段 | 血量 | 内力 | 攻击 | 防御 | 命中 | 闪避 | 暴击 | 速度 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 第 1 年 Boss | 1800-2200 | 550-700 | 95-115 | 55-70 | 75-86 | 8-12 | 10-12 | 1.45-1.60 |
| 第 2 年 Boss | 3200-4200 | 900-1200 | 135-165 | 85-110 | 82-88 | 10-16 | 14-20 | 1.58-1.65 |
| 第 3 年 Boss | 6000-8000 | 1600-2300 | 190-240 | 130-170 | 90-95 | 16-22 | 18-22 | 1.72-1.82 |

Boss 设计边界：

- Boss 可以拥有强机制，但不能无边界无限叠层。
- Boss 的毒、流血、寒气、断筋、蛊都要遵守层数上限和自然衰减。
- 终局 Boss 可以有阶段机制，例如 50% 血净化一次、回血一次、开场护体一次。
- Boss 不应被单一流派完全克死；但每条主线 Boss 可以偏向克制某些玩法。

## 7. 程序实现建议

建议新增数据结构：

```js
DATA.storylines = {
  wanderer: {
    id: "wanderer",
    name: "孤云逐浪",
    threatName: "武盟威势",
    bosses: {
      1: { ... },
      2: { ... },
      3: { ... }
    },
    events: [ ... ]
  },
  constable: {
    id: "constable",
    name: "铁鹰入局",
    threatName: "厂卫疑云",
    bosses: { ... },
    events: [ ... ]
  },
  orthodox: {
    id: "orthodox",
    name: "天衡照邪",
    threatName: "鬼教阴影",
    bosses: { ... },
    events: [ ... ]
  }
};
```

建议修改事件刷新逻辑：

```js
export function refreshEvents(run) {
  const storyEvent = sample(makeStoryEventPool(run), 1)[0];
  const growthEvent = sample(makeGrowthEventPool(run), 1)[0];
  const riskEvent = sample(makeRiskEventPool(run), 1)[0];
  run.events = [storyEvent, growthEvent, riskEvent].filter(Boolean);
}
```

建议修改年末 Boss 逻辑：

- 第 1 年 12 月触发当前角色第 1 年 Boss。
- 第 2 年 12 月触发当前角色第 2 年 Boss。
- 第 3 年 12 月触发当前角色终局 Boss。
- Boss 进入战斗前，根据 `mainThreat` 套用加成。
- 主线情报类事件可写入 `run.storyFlags`，用于削弱 Boss 某些开场机制。

## 9. v0.35 更新

### 9.1 剧情战斗动态奖励
主线偶数月战斗胜利后，除固定 battleReward 外，额外按剩余血量比例动态奖励金钱和经验：
- 公式：`baseReward = floor(敌人血量 × 0.06) × perfMult(血量%) × diffMult`
- 表现系数：`0.4 + 剩余血量% × 0.6`，血量越高奖励越多
- 详见 `IMPLEMENTATION_BALANCE_TABLES.md` 第 9 节

### 9.2 全 Boss 嘴炮补齐
捕快线和正派线的 6 个年终 Boss 补全了 taunt（嘴炮）字段：韩玉阙、沈镇岳、魏承恩、白无咎、桑暮雨、夜摩罗。

### 9.3 散人决心角色页显示
角色属性弹窗（renderCharacterModal）中，孤云线显示散人决心：当前值/10、级别称号、全属性加成百分比。

### 9.4 Toast 堆叠通知
通知系统由替换制改为堆叠制，最多 3 条同时可见，新通知上推旧通知。

### 9.5 升级溢出保护
`gainExp()` 末尾增加 HP/内力上限钳制，防止连续多级升级后当前值超过最大值。
